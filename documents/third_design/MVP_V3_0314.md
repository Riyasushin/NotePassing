# NotePassing MVP V3

> 当前默认产品说明。  
> 接口细节见 `unified_api_contract_v3.md`，未决项见 `DESIGN_TODO.md`。

## 1. 产品定位

NotePassing 是一个基于 BLE 发现的近距离匿名社交 App。

- 蓝牙只负责发现附近设备
- 资料、消息、好友关系、屏蔽都通过服务器传输
- 产品目标不是“在线聊天工具”，而是“现实空间里的轻社交入口”

## 2. 核心闭环

1. 发现附近用户
2. 查看卡片
3. 发起短距聊天
4. 发送好友申请
5. 成为好友并建立长期会话
6. 再次接近时触发 Boost

## 3. 三页结构

### 附近页

- 数据源：本地 `chat_history` 中 active / grace 用户
- 排序：active 好友 > active 非好友 > grace 好友 > grace 非好友
- 卡片内容：头像、昵称、profile 摘要、上一条消息、时间、距离 / 离开时间
- 页面动作：进入聊天、查看资料、发起好友申请

### 好友页

- 数据源：本地 `friend` 列表
- 排序：按最近消息时间
- 卡片附加信息：`meet_count`、当前是否 nearby、Boost 高亮
- 页面动作：进入聊天、查看资料

### 设置页

- 管理昵称、profile、avatar、匿名模式、role_name
- 提供调试入口

## 4. 核心规则

- `chatable = ble_find ∪ heartbeat_receive`
- Server 不管理在线状态，离线判断完全由客户端负责
- 非好友在对方回复前最多发送 2 条消息
- 非好友临时会话在离开蓝牙范围约 1 分钟后过期
- 好友聊天不受距离限制
- block 会从附近、消息、好友申请等链路中统一过滤

## 5. 隐私规则

- 好友始终显示真实身份
- 匿名模式至少对陌生人隐藏头像
- 匿名模式下陌生人看到 `nickname` 还是 `role_name`，由 `DESIGN_TODO.md` 决定

## 6. 客户端 / 服务端边界

### 客户端负责

- BLE 广播与扫描
- `chatable`、active / grace / expired 状态
- 离线检测与 `presence/disconnect` 上报
- 本地缓存、列表排序、消息补漏触发

### 服务端负责

- device / profile 管理
- temp_id 解析
- friend / block / session / message 持久化
- REST API 与 WebSocket 推送
