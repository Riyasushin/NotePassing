# 塞纸条 (NotePassing)

如果我们把社交媒体空间与真实的物理空间同步，会发生什么？
**塞纸条** 是一个 Android App，用于实现 **近距离现实社交网络**。

用户在同一物理空间中，可以通过手机蓝牙发现彼此并即时互动。

核心理念：
> 将现实空间同步到社交网络层，让同一物理空间中的人可以即时互动。

---

# MVP 开发功能

## app功能
1. 进入app，登录页：若第一次进入，根据手机信息生成唯一device_id，并且定义头像、用户名等，上传给服务器；
2. APP分为3页：短距聊天页、好友页、设置页
- 附近页（流动性）：显示所有满足【chatable】条件的用户，通过方形卡片从上到下排序显示
    - 【chatable】条件：（在距离内=（可以扫描到对方||对方扫描到我并且给我更新心跳包））|| 离开距离1min内
    - 卡片类型：CHATABLE且为好友、CHATABLE非好友、CHATABLE失效的好友、CHATABLE失效的非好友、拉黑的卡片直接消失
    - 卡片排序规则： chatable且为好友>chatable非好友>chatable失效的好友>chatable效的非好友，内部根据蓝牙强度排序
    - 卡片内容：文字内容：包括头像、用户名、用户profile前一部分、上一条消息、上一条消息时间、距离||超出距离时间
    - 邀请成为好友窗口：伴随卡片出现申请窗口

- 好友页（固定性）：用户所有的好友
    - 排序规则：发消息早晚
    - 好友卡片额外内容：见面次数，当前是否chatable（boost功能）

- 设置页
    - 修改头像、profile、用户名等
    - 设置在附近页是否显示身份，还是神秘人

# 交互逻辑
## 1. 附近用户发现

客户端通过 **BLE 广播临时加密 ID**。
名片信息包含：
- nickname
- avatar
- profile
(隐藏) - private - state 当前状态是否匿名

---

## 2. 短距聊天

用户可以向蓝牙范围内的用户发送 **短距即时聊天**。

规则：
- 对方未回复前最多发送 **2 条消息**
- 若双方超出蓝牙范围 **1 分钟**，聊天窗口失效

---

## 3. 成为好友

在短距聊天中可以发送 **好友申请**。
双方同意后建立 **好友关系**。

好友之间可以：
- 进行长期聊天
- 不再受蓝牙距离限制

---

## 4. 熟人再次接近
当好友重新进入蓝牙范围时：
- 触发 **Boost 提示**

- 震动一次
- UI 高亮

用于增强线下社交体验。

# 传输原理

参考文件[text](../unified_api_contract.md)
蓝牙仅用于 **发现附近用户**。
所有用户信息与聊天数据均通过服务器传输。

工作流程：
- 名片请求：
1. 客户端 BLE 广播 **临时加密 ID**
2. 其他设备扫描到该 ID
3. 客户端将 ID 上传服务器
4. 服务器返回对应用户信息，显示在卡片上


- 更新设置（和名片请求使用同一个接口）
1. 更新profile、photo、匿名状态与否、nickname

- 消息发送
1. 消息类型1 common：直接对对方发送消息：
判断逻辑：对方chatable||对方为好友
2. 消息类型2 friend request：特殊类型消息，发送并回复好友申请
判断逻辑：发送：对方曾经或现在chatable过；接受：没有block对方
3. 消息类型 heartbeat： 向所有扫描到的对象发送心跳包，5s发送一个

- 给服务器同步信息
1. 同步friend list和block list
