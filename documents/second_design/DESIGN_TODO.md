## TODO-1：附近页好友的匿名展示逻辑

**场景**：用户 A 和用户 B 是好友。B 开启了匿名模式。A 在附近页扫到 B。

**问题**：A 的附近页卡片上，B 应该显示为：
- **(a)** 好友身份（显示真名 nickname + 头像 + 好友角标）—— 匿名对好友无效
- **(b)** 匿名身份（显示 role_name、隐藏头像）—— 附近页统一走匿名规则，不论是否好友

**当前设计**：`§0.5 隐私可见性规则` 写了好友可见所有字段，暗示方案 (a)。但 MVP 描述了"附近页是否显示身份还是神秘人"，未区分好友/非好友。

**建议**：选方案 (a)，附近页好友始终显示真实信息，匿名模式只对陌生人生效。

**状态**：⏳ 待确认

## TODO-2：匿名模式下的名称展示策略

**场景**：用户 B 开启匿名模式，陌生人 A 在附近页看到 B。

**问题**：A 看到的卡片上显示什么名字？
- **(a)** 仍显示 B 的 nickname（当前 API §0.5 定义：nickname 对陌生人可见）
- **(b)** 显示 role_name 替代 nickname（MVP 描述的"神秘人"行为，真名隐藏）
- **(c)** 显示固定文案"神秘人"，role_name 作为副标题

**当前冲突**：
- `§0.5` 写 nickname 对陌生人 ✓ → 方案 (a)
- MVP 描述 "设置在附近页是否显示身份，还是神秘人" → 暗示方案 (b) 或 (c)

**影响范围**：若选 (b)/(c)，需要修改 `§0.5` 可见性表，将 nickname 改为"匿名模式下对陌生人隐藏"，同时 Server 端的 `/presence/resolve` 返回逻辑也需要调整。

**状态**：⏳ 待确认
