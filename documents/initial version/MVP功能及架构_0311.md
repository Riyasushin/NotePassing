# 塞纸条 (NotePassing)

如果我们把社交媒体空间与真实的物理空间同步，会发生什么？

**塞纸条** 是一个安卓 App，定位为“视域内”短距社交。通过手机蓝牙广播应答建立使用者拓扑网络，实现近距离下的即时社交。

适用于：

- 校园社交
- 会议
- 聚会
- 漫展
- 搭讪

核心理念：

> 将现实空间同步到社交网络层，让同一物理空间中的人可以即时互动。

---

# MVP 功能

## 0. 短距广播

用户可选择 **【匿名者】** 或 **【自定义角色】** 名片。

名片包含：

- 昵称
- tag 标签
- 简短 profile

App 通过蓝牙持续广播 **加密 ID**。

用户可以在附近列表中看到：

- 附近用户的名片
- 大致距离

---

## 1. 直接聊天

用户可以向蓝牙广播范围内的人发送 **短距即时聊天**。

规则：

- 对方未回复前最多发送 **2 条消息**
- 如果双方超出蓝牙距离 **1 分钟**，聊天窗口失效

---

## 2. 成为好友

在短距聊天中可以发送 **好友申请**。

双方同意后进入 **超距聊天关系**，身份变为 **完整形象**。

---

## 3. 熟人近距离社交

当好友重新进入蓝牙范围时：

- 出现 **Boost 提示效果**

---

# 升级功能（非 MVP）

- 用户可自定义标签形成小团体，接近时高亮或震动提醒（适用于亚文化社群）
- 用户可以发表公开发言，在他人的扫描窗口中伴随名片显示
- 根据蓝牙拓扑生成临时聊天室或留言板（例如“有人一起吃饭吗”“求组队”等）
- 超距聊天关系形成蓝牙组网图时出现集体 Boost 特效

---

# 技术原理

客户端通过 **蓝牙持续广播加密 ID**。

当用户打开 App 时：

1. 前台扫描附近蓝牙广播
2. 获取附近设备广播的加密 ID
3. 将 ID 上传服务器
4. 服务器返回对应用户的名片信息
5. 建立即时聊天或好友关系

蓝牙仅用于 **近距离发现与拓扑关系判断**。

聊天消息通过 **移动网络服务器** 传输。

# 技术栈

## Android 客户端

语言：

- Kotlin

核心组件：

- Android BLE API（蓝牙广播与扫描）
- WebSocket / HTTP API
- Jetpack Compose（UI）
- Room（本地缓存）
- Kotlin Coroutines（异步）

权限：

- Bluetooth
- Bluetooth Scan
- Bluetooth Advertise
- Nearby Devices

---

## 服务端

语言（任选）：

- Node.js（Express / NestJS）
- Python（FastAPI）

核心服务：

- Profile Service（用户资料）
- Presence Service（附近关系）
- Messaging Service（聊天）
- Relation Service（好友关系）

通信：

- REST API
- WebSocket（即时消息）

数据库：

- PostgreSQL / MySQL

缓存：

- Redis（在线状态与附近关系）

    ├─ Profile API

      ↓

      ↓

      ↓
服务器返回名片

# MVP 目标

实现一个 **现实空间同步的附近社交网络最小闭环**：

```
蓝牙发现附近的人
        ↓
查看名片
        ↓
短距聊天
        ↓
成为好友
        ↓
再次接近触发Boost
```
